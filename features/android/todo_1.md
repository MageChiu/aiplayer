# Android MVP UI 设计

## 1. 设计目标

基于 `features/todo_1.md` 中“阶段 5：Android MVP 设计”的规划，Android 第一版只解决“移动端是否能以较低复杂度跑通核心观看链路”这个问题，不追求桌面版功能等价。

本设计文档的目标是：

- 明确 Android MVP 的功能边界
- 产出可落地的 UI 信息架构与页面设计
- 为 `src/ui/mobile/android` 提供目录与模块划分建议
- 为 `src/platform/mobile` 的 Android 实现预留接口边界
- 为播放器路线与本地翻译路线评估提供 UI 侧输入

## 2. MVP 功能边界

### 2.1 第一版支持

- 本地视频导入与播放
- 内挂字幕或外挂字幕的显示
- 在线翻译字幕
- 基础播放控制
- 基础字幕样式调整
- 基础错误提示与任务进度提示

### 2.2 第一版不支持

- torrent 播放与下载
- 桌面版本地 GGUF helper 可执行文件方案
- 桌面日志窗口
- 复杂模型下载管理 UI
- 多任务批量转写工作台
- 高级调试面板
- 复杂窗口布局与多面板并排模式

### 2.3 MVP 的核心用户路径

第一版只优化以下主链路：

1. 用户进入首页
2. 选择本地视频
3. 进入播放器页面
4. 加载已有字幕或追加外挂字幕
5. 开启在线翻译
6. 在播放过程中查看原文字幕与译文字幕
7. 必要时调整字幕显示、翻译目标语言与播放进度

如果这条链路稳定，Android MVP 就具备初步价值。

## 3. 产品定位与设计原则

### 3.1 产品定位

Android MVP 不是桌面版的“缩小版”，而是围绕移动端场景重组后的“轻量观看工具”：

- 更强调单手操作
- 更强调沉浸式播放
- 更强调对系统权限和系统返回行为的适配
- 更强调异常情况下的可恢复操作

### 3.2 UI 设计原则

- 单任务优先：同一时刻只聚焦一个视频会话
- 播放优先：播放器页面应优先保证观看体验，而不是堆叠设置项
- 渐进暴露：高级设置不在首屏出现，通过底部弹层逐步展开
- 可恢复：文件权限失效、字幕加载失败、网络翻译失败时要给出明确补救动作
- 状态清晰：让用户知道当前是在“仅播放”“有字幕”“翻译中”“翻译失败重试中”
- 移动优先：交互元素尺寸、点击范围、底部手势区域都按 Android 习惯处理

## 4. 信息架构

### 4.1 页面结构

建议 Android MVP 只包含 4 个一级页面 / 场景：

- 启动页 `Launch`
- 首页 `Home`
- 播放页 `Player`
- 设置页 `Settings`

其中真正的核心是：

- 首页负责“选内容”
- 播放页负责“消费内容”

### 4.2 页面关系

```text
Launch
  -> Home
      -> System File Picker
      -> Player
      -> Settings

Player
  -> Subtitle Picker
  -> Translation Sheet
  -> Subtitle Style Sheet
  -> Playback More Sheet
```

### 4.3 导航原则

- 一级导航不建议使用底部 Tab
- Android MVP 更适合单栈导航
- 首页进入播放页后，播放页为沉浸式主场景
- 播放页所有次级操作优先使用 Bottom Sheet、Dialog、系统选择器

原因：

- 播放器是高沉浸页面，不适合让底部 Tab 长期占位
- MVP 页面不多，使用单 Activity + Navigation Compose 即可
- 大多数设置只对当前播放会话生效，用弹层比新页面更高效

## 5. 视觉与交互基线

### 5.1 设计语言

建议使用 Material 3，并做播放器场景定制。

基础风格建议：

- 整体深色优先
- 首页可跟随系统主题
- 播放页默认深色沉浸模式
- 控件圆角适中，不做过度拟物
- 强调对比度，字幕文本优先保证可读性

### 5.2 响应式策略

目标先覆盖手机竖屏，兼容横屏播放，不把平板适配作为 MVP 阻塞项。

建议策略：

- 竖屏：首页与设置页为主
- 横屏：播放器进入主工作形态
- 平板：保持可用即可，不做双栏专门设计

### 5.3 手势策略

- 单击视频区域：显示或隐藏播放控件
- 双击左侧 / 右侧：快退 / 快进
- 横向拖动进度条：精准 seek
- 长按字幕区域：呼出字幕相关操作

不建议在 MVP 中加入过多复杂手势，避免与系统返回、系统全屏手势冲突。

## 6. 核心页面详细设计

### 6.1 启动页 Launch

#### 页面目标

- 完成冷启动初始化
- 检查必要依赖是否可用
- 恢复最近一次播放会话入口

#### 页面内容

- App Logo / 名称
- 初始化进度文案
- 最近会话恢复状态提示

#### 状态分支

- 初始化成功：进入首页
- 最近会话可恢复：弹出“继续播放”提示
- 初始化失败：进入错误状态页或首页降级模式

#### 设计说明

启动页停留时间应尽量短，不承担复杂设置功能。

### 6.2 首页 Home

#### 页面目标

- 让用户快速开始一次视频播放
- 展示最近播放记录
- 提供设置入口

#### 页面结构

1. 顶部栏
2. 主操作区
3. 最近记录区
4. 状态提示区

#### 顶部栏

建议包含：

- 页面标题：`AI Player`
- 设置按钮
- 可选的帮助 / 关于入口

#### 主操作区

建议放置 2 个核心卡片按钮：

- `打开视频`
- `打开字幕`

说明：

- `打开视频` 是绝对主按钮
- `打开字幕` 在未选视频时也可点击，但会提示需在播放页关联或选视频后继续

#### 最近记录区

列表项建议包含：

- 视频名称
- 最近播放时间
- 上次播放进度
- 是否包含字幕 / 翻译配置标识

点击后行为：

- 若文件仍可访问，直接进入播放页并恢复进度
- 若 URI 已失效，提示重新授权或重新选择文件

#### 状态提示区

用于展示：

- 网络不可用提示
- 上次翻译失败提示
- 存储访问权限失效提示

#### 首页交互要求

- 首次进入时，应把“打开视频”作为最强视觉主操作
- 最近记录最多展示 5 到 10 条
- 长列表不作为 MVP 重点，不需要复杂筛选功能

### 6.3 播放页 Player

#### 页面目标

这是 Android MVP 的绝对核心页面，需要同时承载：

- 视频播放
- 字幕展示
- 在线翻译结果展示
- 会话级设置

#### 页面布局

建议采用“视频区域全屏 + 覆盖式控制层”的设计。

页面可以分为 5 层：

1. 视频渲染层
2. 字幕显示层
3. 中央播放控制层
4. 顶部信息栏
5. 底部控制栏

#### 1. 视频渲染层

职责：

- 承载播放器组件
- 适配横竖屏切换
- 显示加载中、错误态占位

需要预留：

- `libmpv` 视图容器或平台播放器视图容器

#### 2. 字幕显示层

建议默认支持三种模式：

- 仅原文
- 原文 + 译文双行
- 仅译文

字幕区域设计要求：

- 默认位于视频下半区域靠近底部，但避开系统手势区和底部控制栏
- 文本最大宽度受控，避免横向过长
- 支持阴影 / 描边 / 半透明背景三种增强可读策略

每条字幕建议结构：

- 原文一行或两行
- 译文一行或两行

双语显示时建议：

- 原文在上，译文在下
- 译文颜色略弱于原文或反过来，需在设计稿中统一
- 当前高亮字幕与历史字幕不要同时长时间停留，避免信息噪声

#### 3. 中央播放控制层

控件显示时机：

- 用户单击视频区域后显示
- 数秒无操作后自动隐藏
- 暂停时保持显示

核心控件：

- 播放 / 暂停
- 快退 10 秒
- 快进 10 秒
- 加载中动画

#### 4. 顶部信息栏

建议内容：

- 返回按钮
- 视频标题
- 可选状态标签：`字幕已加载`、`翻译中`、`离线`
- 更多按钮

交互要求：

- 横屏时标题自动省略
- 标题区不占用过高垂直空间

#### 5. 底部控制栏

建议包含：

- 当前时间 / 总时长
- 进度条
- 字幕按钮
- 翻译按钮
- 样式按钮
- 全屏 / 方向切换按钮

如果空间不足，建议优先保留：

- 进度条
- 字幕按钮
- 翻译按钮
- 播放按钮

### 6.4 播放页子面板设计

#### A. 字幕面板 Subtitle Sheet

用途：

- 管理字幕来源与显示模式

建议包含：

- 当前字幕状态
- 选择外挂字幕文件
- 切换显示模式：原文 / 双语 / 仅译文
- 字幕偏移调整
- 字幕开关

状态设计：

- 未加载字幕
- 已加载字幕
- 加载失败

失败提示需要给出明确操作：

- 重新选择字幕
- 清除当前字幕

#### B. 翻译面板 Translation Sheet

用途：

- 启用或关闭在线翻译
- 调整目标语言
- 展示翻译服务状态

建议字段：

- 翻译开关
- 目标语言
- 翻译服务提供方
- 网络状态提示
- 错误信息与重试入口

MVP 不建议暴露过多高级参数，避免把桌面设置直接搬过来。

#### C. 字幕样式面板 Subtitle Style Sheet

建议配置项：

- 字号
- 文本颜色
- 背景样式
- 底部偏移
- 双语间距

交互原则：

- 所有样式调整都应即时预览
- 支持“恢复默认”

#### D. 更多操作面板 More Sheet

建议放置低频项：

- 播放速度
- 屏幕常亮开关
- 重载视频
- 打开新视频
- 会话信息

## 7. 设置页设计

### 7.1 页面目标

设置页只承载“全局配置”，不要侵入当前会话强相关设置。

### 7.2 建议分组

- 播放设置
- 字幕设置
- 翻译设置
- 存储与缓存
- 关于

### 7.3 推荐配置项

#### 播放设置

- 默认进入全屏
- 双击快进 / 快退时长
- 是否自动恢复上次进度
- 是否保持屏幕常亮

#### 字幕设置

- 默认显示模式
- 默认字号
- 默认字幕位置

#### 翻译设置

- 默认目标语言
- 是否在有网络时自动开启翻译
- 翻译失败时是否自动重试

#### 存储与缓存

- 缓存占用大小
- 清理缓存
- 已授权目录 / 文件访问说明

#### 关于

- App 版本
- 核心组件版本
- 隐私说明

## 8. 关键交互流程

### 8.1 打开本地视频

流程：

1. 首页点击 `打开视频`
2. 调起 Android 系统文件选择器
3. 用户选择视频文件
4. App 持久化保存 URI 访问权限
5. 进入播放页并开始加载
6. 成功后可恢复上次进度

异常分支：

- 用户取消选择：回首页
- 文件无权访问：提示重新授权
- 文件格式不支持：提示错误并允许重新选择

### 8.2 添加外挂字幕

流程：

1. 播放页点击 `字幕`
2. 打开字幕面板
3. 点击 `选择字幕文件`
4. 调起系统文件选择器
5. 解析字幕并加载

异常分支：

- 格式不支持
- 字幕编码异常
- 时间轴解析失败

UI 要求：

- 错误提示必须告诉用户是“无法读取”“格式不支持”还是“解析失败”

### 8.3 开启在线翻译

流程：

1. 播放页点击 `翻译`
2. 打开翻译面板
3. 开启翻译并选择目标语言
4. 系统开始按字幕片段请求在线翻译
5. 成功后在字幕层展示双语

状态要求：

- 首次发起请求时显示轻量进度提示
- 网络慢时不能阻塞视频播放
- 翻译失败时保留原文字幕
- 翻译恢复后无缝刷新译文

### 8.4 恢复最近播放

流程：

1. 首页点击最近记录
2. 校验视频 URI 是否仍有效
3. 恢复播放进度、字幕配置、翻译开关状态

注意：

- 若恢复失败，必须允许用户只恢复视频、不恢复字幕和翻译配置

## 9. UI 状态模型

为了让 UI 和未来 `core` / `platform` 边界清晰，建议在 Android UI 层定义明确的页面状态。

### 9.1 首页状态

- `Idle`
- `LoadingRecentItems`
- `RecentItemsLoaded`
- `Empty`
- `PermissionIssue`
- `Error`

### 9.2 播放页状态

- `Preparing`
- `Ready`
- `Buffering`
- `Paused`
- `Playing`
- `Completed`
- `Error`

### 9.3 字幕状态

- `SubtitleOff`
- `SubtitleLoading`
- `SubtitleReady`
- `SubtitleEmpty`
- `SubtitleError`

### 9.4 翻译状态

- `TranslationOff`
- `TranslationStarting`
- `TranslationActive`
- `TranslationPartialFailure`
- `TranslationError`
- `TranslationNoNetwork`

### 9.5 设计要求

- UI 文案与图标必须能准确映射这些状态
- 页面状态不应直接绑定底层播放器实现类型
- 所有错误状态都需要有明确恢复动作

## 10. 组件设计建议

### 10.1 可复用组件

建议在 `src/ui/mobile/android/components` 下沉淀以下组件：

- `PrimaryActionCard`
- `RecentMediaItem`
- `PlayerOverlayControls`
- `SubtitleTextBlock`
- `StatusChip`
- `SettingGroup`
- `EmptyStateView`
- `ErrorStateView`

### 10.2 组件设计要求

- 输入参数尽量基于纯状态数据
- 避免组件直接拿 platform 层对象
- 尽量让播放器视图容器和普通 Compose 组件解耦

### 10.3 字幕组件细则

`SubtitleTextBlock` 是关键组件，需要重点设计：

- 支持原文 / 译文独立开关
- 支持多行文本
- 支持即时切换样式
- 支持不同播放器容器上的安全区域计算

## 11. Android UI 层目录设计

根据阶段 5 规划，建议目录放在：

- `src/ui/mobile/android`

建议采用如下结构：

```text
src/ui/mobile/android/
  app/
    AndroidAppEntry
    AndroidNavigator
  navigation/
    AppRoutes
    AppNavGraph
  screens/
    launch/
      LaunchScreen
      LaunchViewModel
      LaunchUiState
    home/
      HomeScreen
      HomeViewModel
      HomeUiState
      HomeIntent
    player/
      PlayerScreen
      PlayerViewModel
      PlayerUiState
      PlayerIntent
      sheets/
        SubtitleSheet
        TranslationSheet
        SubtitleStyleSheet
        MoreSheet
    settings/
      SettingsScreen
      SettingsViewModel
      SettingsUiState
  components/
    player/
    subtitle/
    settings/
    common/
  theme/
    Color.kt
    Type.kt
    Theme.kt
  model/
    ui/
      RecentMediaUiModel
      SubtitleUiModel
      TranslationUiModel
  mapper/
    UiStateMapper
```

### 11.1 目录设计原则

- `screen` 内只负责页面编排，不写复杂业务逻辑
- `ViewModel` 通过 service / controller 获取状态
- `mapper` 负责把 `core` / `platform` 数据映射为 UI 模型
- `components` 尽量无业务副作用，方便复用和预览

## 12. Android 平台适配层对 UI 的影响

阶段 5 明确要求设计 Android 平台适配层，以下是与 UI 强关联的点。

### 12.1 存储权限与文件访问

建议优先使用 Android Storage Access Framework，而不是依赖传统文件路径访问。

对 UI 的直接影响：

- “打开视频”与“打开字幕”都应走系统文件选择器
- 最近记录不能只保存文件路径，要保存可恢复的 URI 信息
- 文件失效时需要专门的“重新授权”提示交互

### 12.2 模型缓存目录

虽然 Android MVP 暂不主推本地 GGUF helper，但缓存目录仍然需要标准化。

对 UI 的影响：

- 设置页需要展示缓存占用
- 后续如加入本地模型，应直接复用该设置分组

### 12.3 后台任务

即使 MVP 暂不做复杂下载，仍可能存在：

- 字幕解析
- 会话恢复
- 网络翻译请求

对 UI 的影响：

- 需要轻量进度提示
- 任务失败时需要非阻塞式错误反馈
- 不建议在 MVP 中引入复杂任务中心页面

### 12.4 原生库打包

无论选择 `libmpv` 还是其他播放器方案，UI 需要考虑：

- 首次加载慢的感知处理
- 某些 ABI 不支持时的降级提示
- 原生初始化失败时的错误页设计

## 13. 播放器路线对 UI 的约束

阶段 5 需要评估两条 Android 播放器路线：

- 方案 A：继续 `libmpv`
- 方案 B：平台播放器 + 复用 `core`

### 13.1 方案 A：`libmpv`

优点：

- 更容易接近桌面现有能力模型
- 某些字幕与播放行为可更好复用经验

缺点：

- Android 集成和稳定性成本更高
- 原生层与 Compose UI 的边界更复杂
- 初始化、生命周期、渲染适配难度更大

对 UI 的影响：

- 播放容器可能是嵌入式原生视图
- 需要更谨慎处理全屏、旋转、前后台切换
- 错误态和加载态要设计得更健壮

### 13.2 方案 B：平台播放器

优点：

- 更贴合 Android 生命周期与媒体生态
- 接入成本和系统兼容性通常更好
- UI 和播放器控制层更容易标准化

缺点：

- 与桌面播放器行为的一致性需要重新打磨
- 某些字幕能力可能需要额外补齐

对 UI 的影响：

- Compose + 平台播放器控件的组合通常更简单
- 更适合 MVP 快速验证

### 13.3 UI 侧建议结论

从 UI 落地角度，MVP 更建议优先采用“平台播放器 + 核心能力复用”的路线，原因是：

- 交互集成成本更低
- 生命周期问题更少
- 能更快验证“播放 + 字幕 + 在线翻译”这条主链路

如果未来需要更高一致性的高级播放能力，再评估是否切回 `libmpv`。

## 14. 本地翻译路线对 UI 的约束

阶段 5 明确指出，不继续沿用桌面 helper 可执行文件方案。

### 14.1 MVP 阶段建议

Android 第一版 UI 只围绕“在线翻译”设计，不把“本地翻译”做成显性主功能。

原因：

- 能显著降低 UI 复杂度
- 避免早期暴露模型下载、模型切换、资源不足等复杂状态
- 更符合 MVP 验证目标

### 14.2 为未来进程内模型服务预留

虽然 MVP 不主打本地翻译，但设置与翻译面板应预留可扩展位：

- 翻译模式：在线 / 本地
- 模型状态：未安装 / 安装中 / 可用
- 资源状态：存储不足 / 内存压力高

当前这些入口可以先不展示，只在状态模型中预留。

## 15. 文案与反馈设计

### 15.1 成功反馈

- 视频已打开
- 字幕已加载
- 已切换为双语字幕
- 翻译已开启

### 15.2 失败反馈

建议错误提示要具体，不要只写“失败”：

- 无法访问该文件，请重新选择
- 当前字幕文件格式不受支持
- 网络不可用，已暂停在线翻译
- 翻译服务暂时不可用，请稍后重试
- 播放器初始化失败，请重新打开视频

### 15.3 空状态

- 还没有最近播放记录
- 当前未加载字幕
- 当前未开启翻译

## 16. 无障碍与可用性要求

- 所有核心按钮应提供可访问描述
- 文字与背景对比度满足基础可读性要求
- 字幕字体大小支持较大范围调节
- 控件点击区域不小于 Android 常见可触达标准
- 横屏下也要保证返回与暂停按钮可触达

## 17. MVP 验收标准

如果 Android UI 达到以下标准，可以认为本设计落地合格：

- 用户可从首页稳定打开本地视频
- 用户可在播放页完成播放、暂停、快进、快退、seek
- 用户可加载外挂字幕并正常显示
- 用户可开启在线翻译并看到译文
- 翻译失败不影响原视频继续播放
- 文件权限失效时，UI 可引导用户恢复访问
- 页面状态与错误状态清晰，不依赖桌面式复杂面板
- UI 目录结构可支撑后续继续扩展而不推翻重做

## 18. 推荐实施顺序

建议 Android UI 按以下顺序推进：

1. 先搭建 `src/ui/mobile/android` 基础目录与导航
2. 完成 `HomeScreen` 与系统文件选择流程
3. 接入最小可用 `PlayerScreen`
4. 做字幕显示层与字幕面板
5. 做在线翻译面板与翻译状态展示
6. 做设置页与会话恢复
7. 补齐错误态、空态、权限失效处理

## 19. 结论

Android MVP 的 UI 不应直接复制桌面版结构，而应围绕“本地视频观看 + 字幕显示 + 在线翻译”重新设计成单任务、单栈、沉浸式播放器体验。

从落地角度看，这套设计有三个关键结论：

- 功能上，必须严格控制在 MVP 边界内
- 结构上，应采用 `src/ui/mobile/android` 的分层目录设计
- 技术路线侧，UI 更适合优先配合平台播放器方案，而不是一开始就绑定复杂的 `libmpv` 集成

这样可以在不提前背负桌面历史包袱的前提下，尽快验证 Android 端的核心可用性。
