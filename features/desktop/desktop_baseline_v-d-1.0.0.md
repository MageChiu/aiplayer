# Desktop Baseline v-d-1.0.0

## 目标

这份基线文档用于定义桌面版在跨平台架构改造期间的“不可退化能力”。后续任何涉及 `shared`、`libs`、`platform`、`apps` 的调整，都必须先通过这份基线中的验证项。

## 核心能力基线

### 1. 播放与控制
- 本地视频文件可打开并开始播放
- 支持暂停、停止、重播、拖动进度、倍速切换
- 支持音量调整、静音切换、全屏切换
- 启动参数传入本地文件路径时可自动加载播放

### 2. 输入源
- 支持本地文件打开
- 支持 HTTP/HTTPS 网络流
- 支持 `m3u8` 播放
- 支持 URL 手动输入
- 若构建启用 torrent，支持 `magnet:` 播放；若未启用，必须给出明确错误提示，不能崩溃

### 3. 字幕与 ASR
- 本地文件加载后可执行音频抽取与 ASR
- ASR 成功后可生成时间轴字幕
- 播放过程可按时间同步显示字幕
- seek、暂停、倍速切换后字幕状态不能明显错位

### 4. 翻译
- 支持在线翻译模式
- 支持本地 GGUF 翻译模式
- 修改翻译设置后已加载字幕可重新翻译
- 翻译失败时必须给出可见错误信息

### 5. 模型管理
- 可展示 ASR/翻译模型列表
- 可判断模型是否已安装
- 可下载模型并显示下载状态
- 可打开模型目录

### 6. 设置与状态持久化
- 设置项可保存并在重启后恢复
- 设置项至少覆盖源语言、目标语言、翻译模式、provider、API 配置、本地模型路径
- 设置保存后播放器和字幕链路仍保持可用

### 7. 日志与诊断
- 日志窗口可打开
- 播放、翻译、模型、错误相关状态会进入日志
- 回归验证期间可以通过日志定位失败原因

### 8. 打包与发布
- 桌面版可正常构建
- macOS 包内包含翻译 helper 与运行所需依赖
- 平台打包脚本不能因目录重构而失效

## 最小回归矩阵

### P0 必测
1. 启动应用，主窗口显示正常
2. 打开本地 `mp4`，验证播放、暂停、拖动、音量、全屏
3. 打开设置窗口并保存设置，验证程序无异常
4. 下载或识别一个 ASR 模型，播放本地语音视频，确认生成字幕
5. 启用在线翻译或本地翻译其一，确认双语字幕可显示
6. 打开日志窗口，确认能看到运行状态

### P1 必测
1. 打开 `m3u8` 或普通 HTTP 流
2. 命令行传入本地文件路径启动
3. 切换翻译模式后触发重新翻译
4. 打开模型目录并验证路径可用

### P2 条件测
1. 若构建启用 torrent，验证 `magnet:` 边下边播
2. 若执行打包，验证产物内 helper 和依赖完整

## 当前高风险区

### 1. 主播放链路
- `loadFile` 同时牵涉 mpv、FFmpeg、ASR、字幕、torrent 分支，是最高风险入口

### 2. 字幕同步
- 字幕显示依赖播放时间推进，seek、倍速和暂停后容易发生状态错位

### 3. 设置后重翻译
- 设置保存后会触发全量重翻译，容易引入重复请求、译文丢失或显示延迟

### 4. 本地翻译 helper
- helper 路径依赖打包结构，目录或打包脚本变化容易导致离线翻译失效

### 5. 模型下载与目录管理
- 下载状态、取消、失败回收与目录创建集中在桌面链路，改造时容易破坏既有行为

## 使用规则

- 任何第一批迁移任务开始前，先确认本次改动影响哪些基线能力
- 任何 shared/libs 调整合并前，至少完成一轮 P0 验证
- 如果改动涉及播放主链路、设置链路或打包链路，必须额外覆盖对应 P1/P2 项
