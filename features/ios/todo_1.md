# iOS MVP UI 设计

## 1. 设计目标

基于 `features/todo_1.md` 的阶段 6 规划，iOS 第一版目标不是完整复制桌面版，而是在不依赖桌面实现细节的前提下，先落一个可用的移动端 MVP。

设计目标如下：

- 与 Android MVP 尽量保持一致的能力边界，避免双端产品定义分叉
- 以本地视频播放 + 字幕显示 + 在线翻译为第一优先级
- UI 层只负责交互编排，不直接承载播放器、模型、文件路径等平台细节
- 优先适配 iPhone 竖屏单手操作，再兼容横屏沉浸式播放
- 为后续本地模型、下载管理、更多播放源预留结构，但首版不强行暴露复杂入口

## 2. MVP 功能边界

### 2.1 首版支持

- 本地视频文件导入与播放
- 外挂字幕加载与显示
- 在线翻译开关与目标语言选择
- 基础播放控制：播放、暂停、进度、快进快退、音量、倍速
- 基础字幕控制：显示开关、字号、双语显示、时间偏移
- 播放过程中的错误提示、空状态提示、权限与导入失败反馈
- 基础设置页：播放设置、字幕设置、翻译设置、关于与诊断入口

### 2.2 首版暂不支持

- torrent 播放
- 本地 GGUF helper 可执行文件方案
- 桌面日志窗口式的高级调试界面
- 多任务复杂下载器
- 复杂模型管理界面
- 深度后台长时任务
- iPad 专属多栏交互优化

### 2.3 产品定位

iOS MVP 更像“移动端字幕播放器”而不是“桌面全能力迁移版”。首版聚焦用户最短路径：

1. 导入视频
2. 选择字幕
3. 开启翻译
4. 直接播放

## 3. 设计原则

- 简化层级：首版不做复杂工作台式首页，减少认知负担
- 内容优先：播放器页面优先保证视频、字幕、翻译信息可读
- 状态透明：导入中、加载中、翻译中、失败等状态必须显式反馈
- 渐进暴露：高级能力放入二级面板，不挤占主播放界面
- 平台一致：遵守 iOS 沙盒、文件导入、后台限制与系统交互习惯
- 可扩展：页面结构能平滑接入后续本地模型、下载任务、历史记录

## 4. 信息架构

### 4.1 顶层结构

建议使用 `Tab + NavigationStack + Sheet` 组合：

- `播放`
- `资源`
- `设置`

说明：

- `播放`：承载最近一次播放、空状态引导、进入播放器
- `资源`：承载导入的视频、字幕、最近打开项
- `设置`：承载翻译、字幕、播放、诊断相关配置

如果首版希望更轻量，也可以先只保留两个 Tab：

- `播放`
- `设置`

资源入口通过 `播放` 页和文件导入 sheet 完成。但为了后续扩展，推荐保留 `资源`。

### 4.2 全局导航规则

- 一级页面使用底部 Tab
- 配置类页面使用 `NavigationStack`
- 强交互临时设置使用底部 sheet
- 文件导入使用系统 `Document Picker`
- 权限说明、错误详情、删除确认使用系统弹窗或确认 sheet

## 5. 页面设计

### 5.1 播放页

### 页面目标

让用户在最短时间内恢复最近播放，或开始一次新的播放任务。

### 页面内容

- 顶部导航栏
  - 标题：`播放`
  - 右上角按钮：`导入`
- 最近播放卡片
  - 视频名称
  - 最近进度
  - 上次播放时间
  - 当前字幕状态
  - 当前翻译状态
  - 点击后继续播放
- 快速操作区
  - `导入视频`
  - `加载字幕`
  - `开始播放`
- 最近资源列表
  - 最近打开的视频
  - 最近使用的字幕
  - 失败项目带错误标记
- 空状态
  - 插画或简洁占位
  - 文案说明“导入一个视频开始使用”
  - 主按钮：`导入视频`

### 交互要点

- 如果存在未完成的最近播放，默认突出展示继续播放卡片
- 如果没有任何资源，首屏只保留一个核心 CTA，避免选择过多
- 导入成功后自动询问是否立即选择字幕并进入播放页

### 5.2 资源页

### 页面目标

集中管理用户在 iOS 沙盒中可见的媒体与字幕资源。

### 页面内容

- 分段切换
  - `视频`
  - `字幕`
  - `最近`
- 列表单元信息
  - 名称
  - 文件大小
  - 最近使用时间
  - 状态标签，例如 `可播放`、`缺少字幕`、`导入失败`
- 顶部工具
  - `导入视频`
  - `导入字幕`
- 列表侧滑操作
  - 删除引用
  - 重新关联
  - 查看详情

### 交互要点

- 首版只管理应用已导入或已授权访问的文件引用，不尝试做复杂文件系统浏览器
- 字幕条目支持与某个视频重新关联
- 最近列表用于帮助用户快速恢复上下文

### 5.3 播放器页

### 页面目标

播放器页是 MVP 核心页面，需要同时兼顾视频、字幕、翻译、操作控制与状态反馈。

### 布局建议

竖屏：

- 上半区为视频画面
- 下半区为控制区与字幕设置入口

横屏：

- 视频全屏优先
- 控件覆盖在视频上层，自动隐藏

### 页面结构

- 顶部栏
  - 返回
  - 媒体标题
  - 更多菜单
- 视频区域
  - `AVPlayer` 承载的视频画面
  - 中央播放状态按钮
  - 加载中指示器
  - 错误遮罩
- 字幕展示层
  - 原文字幕
  - 译文字幕
  - 双语模式上下排布
- 底部控制区
  - 播放/暂停
  - 进度条
  - 当前时间 / 总时长
  - 快退 10 秒 / 快进 10 秒
  - 倍速
  - 字幕按钮
  - 翻译按钮
  - 更多设置

### 字幕排版建议

- 单语模式：字幕居中显示，底部安全区上方留白
- 双语模式：原文在上，译文在下
- 最多显示 2 到 3 行，超长内容自动缩放并允许换行
- 默认使用较高对比度的描边或半透明背景，确保视频复杂背景下可读

### 手势建议

- 单击：显示或隐藏控制层
- 双击左侧：快退
- 双击右侧：快进
- 横屏拖动进度条：精确 seek
- 长按倍速按钮：临时 2x 播放，可选做增强项

### 更多菜单

- 选择字幕
- 字幕样式
- 翻译设置
- 播放速度
- 音轨选择
- 重新载入资源
- 查看错误详情

### 5.4 字幕设置 Sheet

### 使用场景

用户在播放过程中快速调整字幕显示，而不离开播放器。

### 配置项

- 字幕显示开关
- 双语显示开关
- 字号
- 字幕位置
- 时间偏移
- 原文颜色 / 译文颜色
- 背景样式

### 交互形式

- 使用底部弹出 sheet
- 实时预览，变更立即作用于当前播放器
- 恢复默认按钮放在底部

### 5.5 翻译设置 Sheet

### 使用场景

用户在播放中快速控制翻译行为。

### 配置项

- 在线翻译开关
- 目标语言
- 翻译显示模式
  - 仅译文
  - 原文 + 译文
- 翻译触发策略
  - 自动
  - 手动刷新
- 网络状态提示

### 交互要点

- 首版默认在线翻译，若网络不可用则清晰提示并允许关闭翻译继续播放
- 不在播放器中暴露复杂 provider 切换，避免 MVP 设置过载

### 5.6 导入流程设计

### 视频导入

入口：

- 播放页 `导入`
- 资源页 `导入视频`
- 空状态按钮

流程：

1. 打开系统文件选择器
2. 用户选择视频文件
3. 应用创建安全访问引用
4. 进入导入中状态
5. 导入成功后进入“是否加载字幕”分支

### 字幕导入

入口：

- 播放器页更多菜单
- 播放页快速操作
- 资源页 `导入字幕`

流程：

1. 打开系统文件选择器
2. 选择字幕文件
3. 自动尝试与当前视频匹配
4. 如果匹配失败，进入手动关联选择

### 5.7 设置页

### 页面结构

- 播放设置
  - 默认倍速
  - 自动续播
  - 横屏自动进入全屏
- 字幕设置
  - 默认字体大小
  - 默认双语模式
  - 默认样式
- 翻译设置
  - 默认目标语言
  - 仅 Wi-Fi 下启用在线翻译
- 存储与资源
  - 已导入文件数量
  - 清理无效引用
- 诊断与关于
  - 版本信息
  - 日志导出入口
  - 问题反馈

### 设计原则

- 只放长期偏好项
- 播放时临时生效的选项优先留在播放器内 sheet

## 6. 核心用户路径

### 6.1 首次使用

1. 用户进入 `播放` 页
2. 看到空状态和 `导入视频`
3. 选择本地视频
4. 成功后引导选择字幕
5. 进入播放器
6. 需要时在字幕/翻译 sheet 中做轻量调整

### 6.2 继续播放

1. 用户进入 `播放` 页
2. 顶部展示最近播放卡片
3. 点击后恢复进度
4. 沿用上次字幕与翻译设置

### 6.3 导入字幕失败

1. 用户选择字幕文件
2. 系统无法自动匹配
3. 弹出关联 sheet
4. 用户选择目标视频
5. 保存映射关系后回到播放器

## 7. 状态设计

### 7.1 空状态

- 无视频：提示导入视频
- 有视频无字幕：提示导入字幕，但不阻塞播放
- 有字幕无翻译：提示可开启在线翻译，但不强制

### 7.2 加载状态

- 文件导入中：显示进度或至少显示不可操作状态
- 视频加载中：播放器中央 loading
- 翻译请求中：字幕区域显示轻量状态，不阻塞播放

### 7.3 错误状态

- 文件格式不支持
- 文件引用失效
- 字幕解析失败
- 网络不可用
- 翻译服务失败
- 播放器初始化失败

错误反馈原则：

- 优先局部提示，不轻易打断整个播放流程
- 错误文案要给出下一步建议，例如“重新导入”“关闭翻译继续播放”

### 7.4 权限与系统限制状态

- 文件访问权限被取消
- 后台后资源被系统回收
- 网络切换导致翻译不可用
- 低电量模式导致部分后台行为受限

## 8. iOS 视觉与交互规范建议

### 8.1 风格方向

- 采用系统原生风格，优先使用 SwiftUI 标准组件
- 颜色风格以深色播放器界面 + 浅色/深色自动适配设置页为主
- 避免桌面式密集面板，改为卡片、sheet、分组列表

### 8.2 关键视觉建议

- 播放器优先暗色背景
- 字幕文字采用高对比白色系，译文可略带弱强调色
- 危险动作如删除资源使用系统红色语义
- 导入、继续播放等主 CTA 使用统一强调色

### 8.3 动效建议

- 控制层显示/隐藏使用短时淡入淡出
- 底部 sheet 使用系统默认弹出效果
- 导入完成和错误提示使用轻量 toast 或 banner

## 9. UI 层目录设计

建议目录：

```text
src/ui/mobile/ios/
  App/
    AiPlayerIOSApp.*
    AppCoordinator.*
  Screens/
    Playback/
    Library/
    Settings/
    Import/
  Components/
    PlayerControls/
    SubtitleOverlay/
    ResourceCard/
    StatusView/
  ViewModels/
    PlaybackViewModel.*
    LibraryViewModel.*
    SettingsViewModel.*
  Models/
    PlaybackItemViewData.*
    SubtitleStyleViewData.*
    TranslationOptionViewData.*
  Theme/
    ColorTokens.*
    Typography.*
    Spacing.*
  Services/
    IOSNavigationService.*
    IOSFileImportService.*
  Preview/
```

目录原则：

- `Screens` 只负责页面组织
- `Components` 承载可复用 UI 组件
- `ViewModels` 负责 UI 状态与 action 转发
- 真正的平台文件访问、路径、后台策略不要直接放进 UI 目录

## 10. UI 与平台层边界

UI 层需要依赖抽象接口，而不是直接操作 iOS 平台细节。

建议边界如下：

- UI 层关心
  - 当前可播放资源
  - 当前字幕状态
  - 当前翻译状态
  - 用户操作事件
- `platform/mobile` 关心
  - 沙盒路径
  - 文件导入
  - 文件引用持久化
  - 后台限制
  - 原生播放器封装
- `core` 关心
  - 字幕解析后的统一数据结构
  - 翻译配置
  - 播放状态模型
  - 错误事件模型

UI 最好只拿到这样的抽象能力：

- `IPlayerController`
- `SubtitleState`
- `TranslationState`
- `AppEvent`
- `IPathProvider`
- `ITranslationBackend`

## 11. 播放器路线对 UI 的影响

### 11.1 优先方案：AVPlayer

原因：

- 更贴合 iOS 系统能力与签名分发要求
- 集成成本低于在 iOS 首版强推 `libmpv`
- 系统播放行为、前后台切换、音频会话更容易对齐

对 UI 的影响：

- 播放器页围绕系统视频渲染能力组织
- 字幕叠加层需要由业务 UI 自己管理
- 某些桌面端播放器高级能力首版不暴露

### 11.2 备选方案：libmpv

只在后续证明 `AVPlayer` 无法满足字幕和能力复用需求时再评估。

对 UI 的影响：

- 能力可能更接近桌面
- 但首版复杂度、签名、包体和稳定性风险更高

## 12. 本地翻译路线对 UI 的影响

阶段 6 已明确不建议继续使用 helper 可执行文件方案。

UI 层的策略应是：

- 首版主打在线翻译
- 本地翻译入口先不在主 UI 中显式暴露
- 后续如果接入进程内模型服务，再在设置页加入“本地翻译实验功能”或模型页

这样可以避免：

- 首版设置页过重
- 用户误以为 iOS 也支持桌面同级别模型管理
- UI 提前绑定尚未稳定的本地模型流程

## 13. 后台与系统限制下的 UI 策略

- 不承诺长时间后台翻译或后台复杂下载
- 进入后台时，如果视频暂停或翻译中断，恢复前台后显式提示状态变化
- 若文件引用在系统层失效，回到前台后给出“重新授权访问”入口
- 若网络变化导致在线翻译中断，只关闭翻译能力，不中断本地播放

## 14. 可演进方向

在不推翻首版结构的情况下，后续可逐步增加：

- 播放历史页
- 模型与下载管理页
- 本地翻译实验入口
- iPad 双栏资源浏览
- 更复杂的字幕样式编辑
- 多播放源支持

## 15. 验收标准

当 iOS UI 设计满足以下条件时，可认为阶段 6 的 UI 设计部分完成：

- 已定义清晰的 MVP 支持与不支持边界
- 已定义首版顶层信息架构与页面层级
- 已定义播放器页、导入流程、设置页、字幕与翻译面板
- 已说明空状态、加载态、错误态、权限态处理方式
- 已给出 `src/ui/mobile/ios` 的推荐目录结构
- 已说明 UI 与 `core`、`platform/mobile` 的职责边界
- 已体现 `AVPlayer` 优先与进程内模型服务优先的阶段性判断

## 16. 建议落地顺序

1. 先定义 iOS UI 所需状态模型与 mock 数据
2. 用静态页面搭出 `播放`、`资源`、`设置` 三个一级页面
3. 实现播放器页与字幕 overlay
4. 接入文件导入与资源列表
5. 接入在线翻译状态与播放器内设置 sheet
6. 最后再补诊断、日志导出、清理无效引用等辅助能力
