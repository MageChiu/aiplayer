# Android AIPlayer Develop Rule

1. 版本号规则为， v-a-1.0.0，其中 v-a 表示Android版本号，1.0.0 表示主版本号.次版本号.小版本号。
2. 针对基础功能的需求和改动，将需求和建议提到“features/core” 目录下，文件名称规则为：`android_issue_v版本号.md`
3. 针对于Android功能自身的下个版本规划，放到“features/android” 目录下，文件名称规则为：`android_todo_v版本号.md`，这里的版本号为下一个小版本
4. 针对一些基础功能，或者是觉得core内的一些规划不是很好的，也可以将需求和建议提到“features/core” 目录下，文件名称规则为：`ios_issue_v版本号.md`
5. Android功能的规划，都在“features/android” 目录下，文件名称规则为：`android_todo_v版本号.md`
