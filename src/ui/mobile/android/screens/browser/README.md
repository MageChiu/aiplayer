# Browser Screen Design

## 页面定位

`Browser` 是 Android MVP 的首个主页面，对应 `features/android/todo_1.md` 中的首页 / 内容浏览入口。

它负责三件事：

- 提供 `打开视频` 主操作
- 展示最近播放内容
- 暴露权限异常、网络异常、恢复播放等轻量状态

## 页面结构

```text
+--------------------------------------------------+
| Top Bar                                          |
| AI Player                             Settings   |
+--------------------------------------------------+
| Hero Action Card                                 |
| [ 打开视频 ]                                     |
| [ 打开字幕 ]                                     |
+--------------------------------------------------+
| Status Banner / Permission Hint                  |
+--------------------------------------------------+
| Recent Media Section                             |
| - 最近播放 1                                     |
| - 最近播放 2                                     |
| - 最近播放 3                                     |
+--------------------------------------------------+
```

## 区块说明

### Top Bar

- 左侧显示产品标题
- 右侧提供设置入口
- 不放低频复杂动作

### Hero Action Card

- `打开视频` 是页面最强视觉入口
- `打开字幕` 是辅助入口
- 首次启动时焦点落在该区域

### Status Banner

- 用于承载权限失效、最近恢复失败、网络不可用等提示
- 仅展示一个最高优先级状态
- 支持一个主动作，例如 `重新授权`、`重试`

### Recent Media Section

每个列表项建议展示：

- 视频标题
- 上次播放进度
- 最近打开时间
- 字幕与翻译状态标识

点击行为：

- 文件可访问时直接恢复播放
- 文件失效时提示重新选择或重新授权

## 交互优先级

1. 打开视频
2. 恢复最近播放
3. 进入设置
4. 处理异常 banner

## 代码映射

当前 Browser 页面骨架对应以下代码：

- `BrowserUiState.kt`：页面状态
- `BrowserIntent.kt`：用户动作
- `BrowserIntentResult.kt`：页面副作用输出
- `BrowserViewModel.kt`：页面状态 reducer
